Quo Vadis Navigation Library

Type-safe, multiplatform navigation for Compose Multiplatform

Get Started View on GitHub

Overview

Quo Vadis (Latin for "Where are you going?") is a comprehensive, type-safe navigation library for Kotlin Multiplatform and Compose Multiplatform. It provides a clean, intuitive API for managing navigation across Android, iOS, Desktop, and Web platforms with zero string-based routing.

Android iOS Desktop (JVM) Web (JS) Web (Wasm)

Key Features

Type-Safe Navigation

Compile-time safety with no string-based routing. Navigate with confidence using sealed classes and type-safe destinations.

Truly Multiplatform

Works seamlessly on Android, iOS, Desktop, and Web. Write once, navigate everywhere.

Modular Architecture

Gray box pattern for feature modules. Build scalable apps with clean separation of concerns.

Direct BackStack Access

Full control over navigation stack. Pop multiple screens, clear to specific destinations, and more.

Deep Link Support

URI-based navigation with pattern matching. Handle deep links elegantly across all platforms.

Predictive Back Navigation

Smooth animated back gestures on Android 13+ and iOS. Modern, native-feeling navigation.

Shared Element Transitions

Material Design shared element transitions with bidirectional support for stunning animations.

MVI Architecture Support

First-class MVI pattern integration. Clean, predictable state management for navigation.

Fully Testable

FakeNavigator included for easy unit testing. Verify navigation behavior without UI.

Quick Example

// 1. Define type-safe destinations
sealed class AppDestination : Destination {
    object Home : AppDestination() {
        override val route = "home"
    }
    
    data class Details(val id: String) : AppDestination() {
        override val route = "details"
        override val arguments = mapOf("id" to id)
    }
}

// 2. Create a navigation graph
val appGraph = navigationGraph("app") {
    startDestination(AppDestination.Home)
    
    destination(AppDestination.Home) { _, navigator ->
        HomeScreen(
            onNavigateToDetails = { id ->
                navigator.navigate(AppDestination.Details(id))
            }
        )
    }
    
    destination(SimpleDestination("details")) { dest, navigator ->
        DetailsScreen(
            itemId = dest.arguments["id"] as String,
            onBack = { navigator.navigateBack() }
        )
    }
}

// 3. Setup navigation in your app
@Composable
fun App() {
    val navigator = rememberNavigator()
    
    LaunchedEffect(Unit) {
        navigator.registerGraph(appGraph)
        navigator.setStartDestination(AppDestination.Home)
    }
    
    GraphNavHost(
        graph = appGraph,
        navigator = navigator,
        defaultTransition = NavigationTransitions.SlideHorizontal
    )
}

Why Quo Vadis?

Built for Modern Apps

Quo Vadis was designed from the ground up for Kotlin Multiplatform and Compose Multiplatform. It embraces modern patterns like MVI, reactive state management with Flow, and type-safe APIs.

No External Dependencies

The library is completely self-contained with no external navigation dependencies. This means smaller app sizes and no conflicts with other libraries.

Beautiful Transitions

Built-in support for shared element transitions, predictive back gestures, and custom animations. Create polished, native-feeling apps with minimal effort.
